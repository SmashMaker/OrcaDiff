# OrcaDiff

Extract and compare Orca Slicer parameters embedded in `.gcode` files. Originally built to recover settings after accidentally overwriting an Orca profile with no backup, but works for any parameter comparison between two sliced files.

## How it works

Orca Slicer writes all slicer settings as comments at the end of every `.gcode` file (`CONFIG_BLOCK`). These two scripts extract and diff that block.

## Usage

**Step 1: extract parameters**
```
python clean.py
```
On first run, creates `source/` and `cleaned/`. Drop your `.gcode` files in `source/`, run again, and pick which file is the reference and which is the one to compare.

Or pass files directly:
```
python clean.py <ref.gcode> <tocompare.gcode>
```

Accepts `.gcode`, `.gcode.ref`, and `.gcode.tocompare` as input.

**Step 2: diff**
```
python diff.py
```
Auto-detects files in `cleaned/`. Prints a coloured table in the terminal and saves `diff.md`.

Or pass files directly:
```
python diff.py <file.gcode.tocompare> <file.gcode.ref>
```

## Output

```
source/     original .gcode files
cleaned/    extracted parameter blocks (.gcode.ref / .gcode.tocompare)
diff.md     diff as a markdown table
```

## Requirements

Python 3.9+, no dependencies. Tested on gcode generated by Orca Slicer 2.3.2.
